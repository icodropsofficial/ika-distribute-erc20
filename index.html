IKA Web — ERC20

+ How to use

This tool helps to distribute ERC20 tokens to a large amount of addresses. Fill in your private key (will only be used to send tokens, we DO NOT HAVE ACCESS TO YOUR KEY). If nonce and balance were't calculated automatically, the private key is invalid.

Enter the contract address (go to the token page at etherscan and copy the value in 'contract' field). Set the gas limit if you know how much do you need (or leave the default otherwise, should suffice (or even be way more than enough) for most contracts). Check the token info carefully. If it didn't appear, the contract address is wrong.

Enter the address (should start with 0x), token amount, gas price (gwei) for each transaction. The addresses MUST NOT repeat: only one transaction to a single address.

Check the checkbox and press 'fire!'. Wait for clock icons to get replaced with transactions' status. You can hover over a cross to see why the transaction has failed, or click the successful transaction icon to see the transaction on Etherscan. Note that Etherscan sees if a transaction is confirmed a bit later, so don't get scared if you see a green tick and 'Pending' on Etherscan.

In some (very rare) cases a transaction may get confirmed, but then fall out of the main chain if a fork occurs, so a tick means that a transaction will almost definitely stay in the chain.
